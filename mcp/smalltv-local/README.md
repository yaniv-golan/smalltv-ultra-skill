# SmallTV Local MCP Server (mcp-bash)

This is a local MCP server project built with [`mcp-bash`](https://github.com/yaniv-golan/mcp-bash-framework) that lets Claude or any MCP-capable app/client call SmallTV endpoints from your machine.

## Recommended Install (MCPB)

1. Download the latest `.mcpb` from [GitHub Releases](https://github.com/yaniv-golan/smalltv-ultra-skill/releases/latest).
2. Install the bundle in your MCPB-compatible client (for example, Claude Desktop).
4. Enter your device IP when prompted by the client user-config UI.

This bundle embeds the `mcp-bash` runtime, so end users do not need to install `mcp-bash` separately.

## If IP Prompt Is Not Shown

Some clients may not surface MCPB user-config UI yet. In that case, set:

```bash
export SMALLTV_DEVICE_IP=192.168.5.253
```

## Developer Setup (Optional, Non-MCPB)

Use this only if you want to run/configure the project directly with local `mcp-bash`:

```bash
cd mcp/smalltv-local
./setup.sh --install-mcp-bash --client cursor --device-ip 192.168.5.253
```

To regenerate the bundle after changes:

```bash
# 1. Regenerate MCP resource docs from skill references
python3 ../../scripts/sync_skill_docs_to_mcp.py

# 2. Build the bundle
cd mcp/smalltv-local
mcp-bash bundle
```

## Tools

- `smalltv-get-device-info`
  - Verify device connectivity and identity (call this first)
  - Reads `GET /v.json`
  - Optional arg: `device_ip` (falls back to `SMALLTV_DEVICE_IP`)

- `smalltv-http-request`
  - Call any SmallTV endpoint (`/set`, `/space.json`, `/doUpload`, etc.)
  - Use after verifying device with `smalltv-get-device-info`
  - Args:
    - `path` (required, must start with `/`)
    - `method` (default `GET`)
    - `body` (optional raw body)
    - `content_type` (default `application/json`)
    - `device_ip` (optional, fallback to env var)

## Server Instructions

The file `server.d/server.instructions.md` is sent to MCP clients
automatically during the `initialize` handshake (requires mcp-bash >= 1.1.3).
It contains tool-selection guidance, safety defaults, common endpoint
patterns, and pointers to the MCP resources below.

## MCP Resources and Prompt

The server exposes MCP resources and a prompt for model discoverability:

- **`smalltv-api-reference`** — full HTTP API reference (generated from skill docs)
- **`smalltv-safety-guide`** — safety rules, destructive endpoints, device constraints (generated from skill docs)
- **`prompt.smalltv-get-started`** — quick-start prompt for MCP clients

## Documentation Architecture

Skill `references/` files are the **single source of truth**. MCP resource docs under `resources/` are auto-generated by [`scripts/sync_skill_docs_to_mcp.py`](../../scripts/sync_skill_docs_to_mcp.py) and must not be edited manually. CI enforces freshness.

## Safety

This server can call destructive endpoints if requested (for example reset/reboot/delete/clear). Use carefully on trusted prompts only.

Versioning and MCP tag/release conventions are documented in [`VERSIONING.md`](../../VERSIONING.md).
